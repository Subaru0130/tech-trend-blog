# 🔰 サイト管理・運営ガイド

このドキュメントでは、「今どこで何が起きているのか」「どうやってサイトを見るのか」を分かりやすく解説します。

## 1. サイトを見る方法（2種類あります）

このプロジェクトには、**「自分だけが見れる確認用」**と**「世界中に公開される本番用」**の2つの状態があります。

### 🅰️ 確認用（ローカル環境）
あなたのパソコンの中でだけ動いている状態です。記事を作ったり、デザインを直したりするときはここを見ます。
- **URL**: `http://localhost:3000`
- **使い方**:
    1. ターミナルで `npm run dev` と入力してエンター。
    2. ブラウザで上記のURLを開く。
    3. 終わるときはターミナルで `Ctrl + C` を押す。

### 🅱️ 本番用（公開環境 / Vercel）
世界中の人がアクセスできるインターネット上のサイトです。Amazonの審査にはこのURLを使います。
- **URL**: Vercelのダッシュボードに表示されている `https://tech-trend-blog-xxxx.vercel.app` のようなURL。
- **更新方法**:
    1. パソコンで作業をする。
    2. Gitで「保存（コミット）」して「送信（プッシュ）」する。
    3. 自動的にVercelが更新されます（数分かかります）。

## 2. ツールたちの役割（何のためにあるの？）

| ツール名 | 役割 | 例えるなら... |
| :--- | :--- | :--- |
| **VS Code** | 作業場 | キッチン（ここで料理を作ります） |
| **Next.js** | サイトの骨組み | レシピ兼調理器具（これのおかげで高機能なサイトが作れます） |
| **Git** | 記録・保存 | セーブポイント（失敗しても過去の状態に戻せます） |
| **GitHub** | 倉庫 | 銀行の貸金庫（大切なコードをネット上に保管します） |
| **Vercel** | 公開場所 | レストランの店舗（ここにお客さんが来ます） |

---

## 3. 記事の作り方（自動生成）

現在は「スクリプト」というプログラムを使って記事を自動で作っています。

1. **命令する**: `node scripts/generate-post.mjs` を実行。
2. **作られる**: AIが文章と画像を考えて、`content/posts` フォルダにファイルを作ります。
3. **確認する**: `http://localhost:3000` を見て、変なところがないかチェックします。
4. **公開する**: Gitでプッシュして、本番サイトに反映させます。

---

## 困ったときは？

- **「サイトが表示されない」**: `npm run dev` が動いているか確認してください。
- **「画像が出ない」**: 画像ファイルが `public/images` にあるか確認してください。
